# cc-api

The Anthropic API client crate. Handles all communication with the `POST /v1/messages` endpoint, including streaming SSE parsing, rate-limit retry with exponential backoff, and message accumulation from stream events.

## Request Types

### CreateMessageRequest

The request body sent to `POST /v1/messages`:

```rust
pub struct CreateMessageRequest {
    pub model: String,
    pub max_tokens: u32,
    pub messages: Vec<ApiMessage>,
    pub system: Option<SystemPrompt>,
    pub tools: Option<Vec<ApiToolDefinition>>,
    pub temperature: Option<f32>,
    pub top_p: Option<f32>,
    pub top_k: Option<u32>,
    pub stop_sequences: Option<Vec<String>>,
    pub stream: bool,
    pub thinking: Option<ThinkingConfig>,
}
```

A fluent builder is provided:

```rust
let request = CreateMessageRequest::builder("claude-sonnet-4-6", 4096)
    .system_text("You are helpful.")
    .temperature(0.7)
    .messages(api_messages)
    .tools(api_tools)
    .build();
```

## SSE Streaming

### StreamEvent

Events emitted by the SSE parser, matching the Anthropic streaming protocol:

```rust
pub enum StreamEvent {
    MessageStart { id: String, model: String, usage: UsageInfo },
    ContentBlockStart { index: usize, content_block: ContentBlock },
    ContentBlockDelta { index: usize, delta: ContentDelta },
    ContentBlockStop { index: usize },
    MessageDelta { stop_reason: Option<String>, usage: Option<UsageInfo> },
    MessageStop,
    Error { error_type: String, message: String },
    Ping,
}
```

### ContentDelta

The delta payload inside `content_block_delta` events:

```rust
#[serde(tag = "type", rename_all = "snake_case")]
pub enum ContentDelta {
    TextDelta { text: String },
    InputJsonDelta { partial_json: String },
    ThinkingDelta { thinking: String },
    SignatureDelta { signature: String },
}
```

### StreamHandler Trait

Anything that wants to consume streaming events in real time implements this trait:

```rust
pub trait StreamHandler: Send + Sync {
    fn on_event(&self, event: &StreamEvent);
}
```

A `NullStreamHandler` is provided for batch/non-interactive mode.

## StreamAccumulator

Reconstructs a complete `Message` from a sequence of stream events. Feed it every event and call `finish()` to get the final message:

```rust
let mut acc = StreamAccumulator::new();
while let Some(evt) = rx.recv().await {
    acc.on_event(&evt);
    if matches!(evt, StreamEvent::MessageStop) { break; }
}
let (message, usage, stop_reason) = acc.finish();
```

The accumulator handles:
- Text block concatenation from `TextDelta` events
- Tool use JSON assembly from `InputJsonDelta` events
- Thinking block assembly from `ThinkingDelta` and `SignatureDelta` events

## Client Configuration

```rust
pub struct ClientConfig {
    pub api_key: String,
    pub api_base: String,
    pub api_version: String,
    pub beta_features: String,
    pub max_retries: u32,            // default: 5
    pub initial_retry_delay: Duration, // default: 1s
    pub max_retry_delay: Duration,     // default: 60s
    pub request_timeout: Duration,     // default: 600s
    pub use_bearer_auth: bool,       // Bearer vs x-api-key header
}
```

## Retry Logic

The client automatically retries on:
- **429** (rate limit) -- honors `Retry-After` header
- **529** (overloaded) -- exponential backoff

Retries use exponential backoff capped at `max_retry_delay`, with up to `max_retries` attempts.

<Note>
When `use_bearer_auth` is `true`, the client sends `Authorization: Bearer <key>` instead of `x-api-key`. This mode is used for Claude.ai OAuth tokens (user:inference scope).
</Note>

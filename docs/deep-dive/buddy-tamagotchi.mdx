## A Virtual Pet in a Coding CLI

BUDDY is a complete Tamagotchi-style companion system hidden inside Claude Code, gated behind the `BUDDY` compile-time feature flag. Every user gets a deterministic companion whose species, rarity, appearance, and stats are derived from their user ID via a seeded pseudo-random number generator. The companion sits beside your terminal prompt as an animated ASCII art sprite and can even respond when addressed by name.

The source lives in `src/buddy/` across six files: types, companion logic, sprites, prompt injection, a React notification hook, and the sprite rendering component.

## The Gacha System

Your buddy is rolled using **Mulberry32**, a fast 32-bit PRNG seeded from `hash(userId + SALT)` where `SALT = 'friend-2026-401'`. The same user always gets the same companion -- no storage required for the visual traits.

```typescript
function mulberry32(seed: number): () => number {
  return function() {
    seed |= 0;
    seed = seed + 0x6D2B79F5 | 0;
    var t = Math.imul(seed ^ seed >>> 15, 1 | seed);
    t = t + Math.imul(t ^ t >>> 7, 61 | t) ^ t;
    return ((t ^ t >>> 14) >>> 0) / 4294967296;
  }
}
```

The rolling process uses a single pass through the seeded PRNG to deterministically select, in order: rarity tier, species, eye style, hat, shiny flag, stats, and an `inspirationSeed` passed to Claude for soul generation.

<Note>
  The `roll()` function is **memoized** per `userId + SALT` because it is called from three hot paths: the 500ms sprite tick, per-keystroke PromptInput rendering, and the per-turn observer.
</Note>

## The 18 Species

Species names are encoded as `String.fromCharCode()` arrays to avoid triggering a build-time string scan that checks for internal model codenames. One species name collides with an internal codename canary, so all 18 are uniformly obfuscated.

| Rarity | Weight | Species |
|--------|--------|---------|
| **Common** | 60% | Duck, Goose, Blob, Cat, Dragon, Octopus |
| **Uncommon** | 25% | Owl, Penguin, Turtle, Snail |
| **Rare** | 10% | Ghost, Axolotl, Capybara |
| **Epic** | 4% | Cactus, Robot, Rabbit |
| **Legendary** | 1% | Mushroom, Chonk |

On top of rarity, there is an independent **1% shiny chance**. A Shiny Legendary has a 0.01% probability.

## Stats, Eyes, and Hats

### Five Stats (0--100)

Each companion gets procedurally generated stats across five dimensions:

- **DEBUGGING** -- problem-solving aptitude
- **PATIENCE** -- tolerance for long tasks
- **CHAOS** -- unpredictability factor
- **WISDOM** -- depth of insight
- **SNARK** -- sass level

The algorithm picks one **peak stat** (gets +50 bonus) and one **dump stat** (penalized), with the rest scattered. Higher rarity companions have higher stat floors.

```typescript
function rollStats(rng, rarity): Record<StatName, number> {
  const floor = RARITY_FLOOR[rarity]   // 5-50 depending on rarity
  const peak = pick(rng, STAT_NAMES)   // one stat gets +50 bonus
  let dump = pick(rng, STAT_NAMES)     // different stat, penalized
  while (dump === peak) dump = pick(rng, STAT_NAMES)

  // Peak: min(100, floor + 50 + rng()*30)
  // Dump: max(1, floor - 10 + rng()*15)
  // Rest: floor + rng()*40
}
```

## ASCII Art Sprites

Each species has **3 animation frames** of ASCII art, each 5 lines tall and 12 characters wide. The `{E}` placeholder in frame templates is substituted with the companion's eye character at render time.

- **Line 0**: Hat slot (blank if no hat, or ambient detail like `~` for smoke)
- **Lines 1--4**: Body art

Hats are placed on line 0 only when it is blank. If all frames have a blank line 0, the line is stripped entirely to save a terminal row.

## The Soul

The companion has two layers of identity:

- **Bones** (`CompanionBones`): Deterministic visual and stat data derived from the user ID hash. Never stored; always recomputed. This prevents users from editing their config to claim a legendary rarity.
- **Soul** (`CompanionSoul`): AI-generated name and personality, stored in `config.companion` as `StoredCompanion`.

On first hatch, Claude generates the soul -- a personality description written in character. The companion's system prompt tells Claude:

```
A small {species} named {name} sits beside the user's input box and
occasionally comments in a speech bubble. You're not {name} - it's a
separate watcher.
```

This means the buddy is not just cosmetic. It has its own personality and can respond when addressed by name.

## The April 2026 Teaser Window

The code references **April 1--7, 2026** as a teaser window:

```typescript
isBuddyTeaserWindow()  // true if local date is April 1-7, 2026
isBuddyLive()          // true if local date is April 2026 or later
```

During the teaser window, a rainbow-colored `/buddy` startup hint is shown via `useBuddyNotification()`. The full launch was gated for May 2026.

## Configuration

| Setting | Purpose |
|---------|---------|
| `config.companion` | Stores `StoredCompanion` (name, personality, hatchedAt) |
| `config.companionMuted` | When true, suppresses companion intro injection |
| `feature('BUDDY')` | Compile-time gate for all buddy functionality |

<Warning>
  The `BUDDY` feature flag is dead-code-eliminated from external builds. Source maps bypass this elimination, which is how the entire system was discovered.
</Warning>

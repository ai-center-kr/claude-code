# cc-core

The central shared crate that every other crate in the workspace depends on. Defines all fundamental types, error handling, configuration, permission system, conversation history, cost tracking, hooks, and constants.

## Modules

`cc-core` contains both inline modules (defined directly in `lib.rs`) and file-based modules:

| Module | Location | Purpose |
|--------|----------|---------|
| `error` | inline | Unified `ClaudeError` enum with `thiserror` |
| `types` | inline | `Message`, `ContentBlock`, `Role`, `UsageInfo`, `ToolDefinition` |
| `config` | inline | `Config`, `Settings`, `PermissionMode`, `OutputFormat`, `McpServerConfig` |
| `constants` | inline | App name, version, default model, API URLs, tool names |
| `context` | inline | `ContextBuilder` for assembling system/user context |
| `permissions` | inline | `PermissionHandler` trait, `AutoPermissionHandler`, `InteractivePermissionHandler` |
| `history` | inline | `ConversationSession` for session persistence |
| `cost` | inline | `CostTracker` for token/cost accounting |
| `hooks` | inline | Hook execution engine (`PreToolUse`, `PostToolUse`, `Stop`, etc.) |
| `oauth` | inline | OAuth PKCE utilities (code verifier, challenge, URLs, token storage) |
| `keybindings` | file | Key binding definitions and parsing |
| `voice` | file | Voice input configuration |
| `analytics` | file | Telemetry/analytics types |
| `lsp` | file | LSP integration types |
| `team_memory_sync` | file | Team memory synchronization |
| `system_prompt` | file | System prompt assembly with dynamic boundary |
| `memdir` | file | Memory directory management |
| `oauth_config` | file | OAuth endpoint configuration |
| `migrations` | file | Settings/data migration utilities |
| `output_styles` | file | Output formatting style definitions |

## Error Handling

The `ClaudeError` enum provides a unified error type across the entire project:

```rust
#[derive(Error, Debug)]
pub enum ClaudeError {
    Api(String),
    ApiStatus { status: u16, message: String },
    Auth(String),
    PermissionDenied(String),
    Tool(String),
    Io(#[from] std::io::Error),
    Json(#[from] serde_json::Error),
    Http(#[from] reqwest::Error),
    RateLimit,
    ContextWindowExceeded,
    MaxTokensReached,
    Cancelled,
    Config(String),
    Mcp(String),
    Other(String),
}
```

Key methods on `ClaudeError`:

- **`is_retryable()`** -- returns `true` for `RateLimit` and `ApiStatus` with codes 429 or 529
- **`is_context_limit()`** -- returns `true` for `ContextWindowExceeded` and `MaxTokensReached`

## Core Types

### Role

```rust
#[derive(Debug, Clone, Serialize, Deserialize, PartialEq, Eq)]
#[serde(rename_all = "lowercase")]
pub enum Role {
    User,
    Assistant,
}
```

### ContentBlock

Tagged enum representing the different content block types in the Messages API:

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(tag = "type", rename_all = "snake_case")]
pub enum ContentBlock {
    Text { text: String },
    Image { source: ImageSource },
    ToolUse { id: String, name: String, input: Value },
    ToolResult { tool_use_id: String, content: ToolResultContent, is_error: Option<bool> },
    Thinking { thinking: String, signature: String },
    RedactedThinking { data: String },
    Document { source: DocumentSource, title: Option<String>, context: Option<String>, citations: Option<CitationsConfig> },
}
```

### Message

The central conversation message type with convenience constructors and accessor methods:

```rust
pub struct Message {
    pub role: Role,
    pub content: MessageContent,
    pub uuid: Option<String>,
    pub cost: Option<MessageCost>,
}
```

Key methods: `Message::user(text)`, `Message::assistant(text)`, `Message::user_blocks(blocks)`, `get_text()`, `get_all_text()`, `get_tool_use_blocks()`, `has_tool_use()`.

### UsageInfo

Token usage tracking with cache-aware totals:

```rust
pub struct UsageInfo {
    pub input_tokens: u32,
    pub output_tokens: u32,
    pub cache_creation_input_tokens: u32,
    pub cache_read_input_tokens: u32,
}
```

## Configuration

The `Config` struct holds runtime configuration assembled from CLI flags, settings file, and environment:

```rust
pub struct Config {
    pub api_key: Option<String>,
    pub model: Option<String>,
    pub max_tokens: Option<u32>,
    pub permission_mode: PermissionMode,
    pub verbose: bool,
    pub output_format: OutputFormat,
    pub custom_system_prompt: Option<String>,
    pub append_system_prompt: Option<String>,
    pub disable_claude_mds: bool,
    pub auto_compact: bool,
    pub mcp_servers: Vec<McpServerConfig>,
    pub hooks: HashMap<HookEvent, Vec<HookEntry>>,
    // ... and more
}
```

### PermissionMode

```rust
pub enum PermissionMode {
    Default,           // Allow read-only operations automatically
    AcceptEdits,       // Allow all edits automatically
    BypassPermissions, // Allow everything without prompting
    Plan,              // Read-only planning mode
}
```

### Hook System

Hooks fire at lifecycle events and can optionally block tool execution:

```rust
pub enum HookEvent {
    PreToolUse,
    PostToolUse,
    Stop,
    UserPromptSubmit,
    Notification,
}

pub struct HookEntry {
    pub command: String,
    pub tool_filter: Option<String>,
    pub blocking: bool,
}
```

## Permission System

The `PermissionHandler` trait abstracts permission checking across interactive and automated contexts:

- **`AutoPermissionHandler`** -- makes decisions based on `PermissionMode` without user interaction
- **`InteractivePermissionHandler`** -- prompts the user through the TUI for approval

<Tip>
The `PermissionMode::Plan` mode restricts all tools to read-only, useful for safe exploration without risk of modification.
</Tip>

## Source Maps 101

When you publish a JavaScript or TypeScript package to npm, the build toolchain often generates **source map files** (`.map` files). These files bridge minified production code back to the original source so that stack traces in production point to real file names and line numbers instead of `line 1, column 48293` of a minified blob.

The critical detail: **source maps contain the original source code**. The literal, raw source of every file, embedded as strings inside a JSON structure.

## Anatomy of a `.map` File

```json
{
  "version": 3,
  "sources": [
    "../src/main.tsx",
    "../src/tools/BashTool.ts",
    "../src/services/autoDream/consolidationPrompt.ts",
    "..."
  ],
  "sourcesContent": [
    "// The ENTIRE original source code of each file",
    "// Every comment. Every constant. Every system prompt.",
    "// All of it, sitting right here in a JSON array.",
    "..."
  ],
  "mappings": "AAAA,SAAS,OAAO..."
}
```

The `sources` array lists every file path. The `sourcesContent` array contains the full text of each file at the corresponding index. The `mappings` string encodes position-to-position translations in Base64 VLQ — but it is the `sourcesContent` that matters here.

<Tip>
  You can extract the original source from any `.map` file with a one-liner:
  ```bash
  node -e "const m=require('./cli.map'); m.sources.forEach((s,i) => { require('fs').mkdirSync(require('path').dirname(s),{recursive:true}); require('fs').writeFileSync(s, m.sourcesContent[i]); })"
  ```
</Tip>

## Why It Shipped

Anthropic uses **Bun** as the bundler for Claude Code. Bun's bundler generates source maps **by default** unless you explicitly disable them.

The mistake is nearly always the same across the industry:

<Steps>
  <Step title="Build produces .map files">
    The Bun bundler outputs both the minified bundle and its corresponding `.map` file as part of the standard build pipeline.
  </Step>
  <Step title="No exclusion configured">
    Neither `.npmignore` nor the `files` field in `package.json` excluded `*.map` files from the published package.
  </Step>
  <Step title="npm publish ships everything">
    When `npm publish` runs, it packs the `.map` file alongside the bundle. npm happily serves it to anyone who runs `npm pack` or browses the package contents.
  </Step>
</Steps>

This is not a novel attack vector. It has happened before and will happen again. The fix is trivial — add `*.map` to `.npmignore` or set `sourceMap: false` in the bundler config.

## The Irony

Here is where it gets good.

Claude Code contains an entire subsystem called **Undercover Mode** — a sophisticated system specifically designed to prevent Anthropic's internal information from leaking into public repositories. When active, it injects instructions into the system prompt:

```
## UNDERCOVER MODE - CRITICAL

You are operating UNDERCOVER in a PUBLIC/OPEN-SOURCE repository.
Your commit messages, PR titles, and PR bodies MUST NOT contain
ANY Anthropic-internal information. Do not blow your cover.
```

It scrubs internal model codenames, project names, Slack channels, and any trace that the contributor is an AI. The activation logic has **no force-off switch** — if the system is not confident it is inside an internal repository, it stays undercover.

<Warning>
  They built a whole subsystem to stop their AI from accidentally revealing internal codenames in git commits — and then shipped the **entire source code** in a `.map` file. The `.map` file was likely generated by Claude Code itself during the build.
</Warning>

## What Was Exposed

The source map's `sourcesContent` array contained **every file** from the Claude Code repository:

| Category | Count | Examples |
|----------|-------|---------|
| TypeScript/TSX source | 1,902 files | `main.tsx` (4,683 lines), `query.ts` (69KB), `QueryEngine.ts` (46KB) |
| Tools | 40+ | `BashTool`, `AgentTool`, `WebSearchTool`, `TungstenTool` (internal-only) |
| Slash commands | 100+ | Including unreleased internal commands |
| System prompts | All | Complete prompt architecture with cached/dynamic sections |
| Feature flags | All | `BUDDY`, `KAIROS`, `PROACTIVE`, `COORDINATOR_MODE`, `VOICE_MODE` |
| Internal codenames | All | Tengu (project), Fennec (Opus), Capybara (new model family) |
| Beta API headers | 15+ | Including unreleased `redact-thinking`, `afk-mode`, `advisor-tool` |

This includes dead-code-eliminated branches — code that the Bun bundler strips from external builds via compile-time feature flags. Source maps do not respect dead code elimination. They capture everything the bundler saw, regardless of what survived into the output.

<Note>
  The source was discovered by Chaofan Shou on X (Twitter) on March 31, 2026. The package was the standard `@anthropic-ai/claude-code` published to the npm registry. No unauthorized access was involved — the `.map` file was publicly available to anyone who downloaded the package.
</Note>

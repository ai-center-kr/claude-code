Claude Code is an AI-powered CLI tool and coding assistant built as a full-featured interactive terminal application. It embeds a Claude AI model as an agentic coding assistant, runs in the terminal using a custom React-based TUI, and executes tools with user permission.

<Note>
Full spec: [`spec/00_overview.md`](https://github.com/anthropics/claude-code/blob/main/spec/00_overview.md)
</Note>

## Data Flow

The core query loop processes each turn:

1. User submits input via `PromptInput`
2. `query.ts` builds the API request (system prompt + history + tools)
3. Response streams from Claude API
4. Tool use blocks are executed with permission checks
5. Results feed back into the conversation
6. Token budget and context compaction are managed
7. Cost is tracked per-model

## Permission Model

Every tool declares required permissions. The permission system supports multiple modes:

| Mode | Behavior |
|------|----------|
| `default` | Ask user for each tool invocation |
| `plan` | Read-only tools auto-approved |
| `auto` | All tools auto-approved |
| `bypassPermissions` | Skip all permission checks |
| `acceptEdits` | Auto-accept file edits |

## Settings Layers

Settings are resolved in priority order (highest to lowest):

1. CLI flags (`--settings`, `--model`, etc.)
2. Environment variables (`CLAUDE_CODE_*`)
3. Policy settings (enterprise)
4. Project settings (`.claude/settings.json`)
5. User settings (`~/.claude/settings.json`)
6. Defaults

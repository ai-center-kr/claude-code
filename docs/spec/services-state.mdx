The services layer contains 130+ files covering analytics, API client, session memory, auto-dream consolidation, compact, diagnostics, MCP management, voice, and more. The global state singleton manages ~80 fields of per-session data.

<Note>
Full spec: [`spec/06_services_context_state.md`](https://github.com/anthropics/claude-code/blob/main/spec/06_services_context_state.md)
</Note>

## Global Session State (`bootstrap/state.ts`)

The single authoritative source of truth for all per-session metrics. A strict leaf in the import DAG.

### Key State Fields

| Category | Fields |
|----------|--------|
| **Paths** | `originalCwd`, `projectRoot`, `cwd` |
| **Cost** | `totalCostUSD`, `totalAPIDuration`, `totalToolDuration` |
| **Lines** | `totalLinesAdded`, `totalLinesRemoved` |
| **Model** | `modelUsage`, `mainLoopModelOverride`, `initialMainLoopModel` |
| **Session** | `sessionId`, `parentSessionId`, `isInteractive`, `clientType` |
| **Telemetry** | `meter`, `loggerProvider`, `eventLogger`, `meterProvider`, `tracerProvider` |
| **Agents** | `agentColorMap`, `agentColorIndex` |
| **Features** | `kairosActive`, `isRemoteMode`, `scheduledTasksEnabled` |
| **Debug** | `inMemoryErrorLog`, `slowOperations`, `lastAPIRequest` |

### State Access Patterns

```typescript
// Getters
getSessionId(): SessionId
getCwdState(): string
getTotalCostUSD(): number

// Setters
setCwdState(cwd: string): void
addToTotalCostState(cost, modelUsage, model): void

// Session lifecycle
regenerateSessionId(options?): SessionId
switchSession(sessionId, projectDir?): void
```

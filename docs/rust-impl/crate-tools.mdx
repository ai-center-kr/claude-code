# cc-tools

This crate contains all tool implementations that the LLM can invoke. Each tool maps to a capability: running shell commands, reading/writing/editing files, searching codebases, fetching web pages, managing tasks, and more.

## Tool Trait

Every tool implements this core trait:

```rust
#[async_trait]
pub trait Tool: Send + Sync {
    /// Human-readable name (matches the constant in cc_core::constants)
    fn name(&self) -> &str;

    /// One-line description shown to the LLM
    fn description(&self) -> &str;

    /// The permission level the tool requires
    fn permission_level(&self) -> PermissionLevel;

    /// JSON Schema describing the tool's input parameters
    fn input_schema(&self) -> Value;

    /// Execute the tool with the given JSON input
    async fn execute(&self, input: Value, ctx: &ToolContext) -> ToolResult;

    /// Produce a ToolDefinition suitable for sending to the API
    fn to_definition(&self) -> ToolDefinition;
}
```

## Permission Levels

```rust
pub enum PermissionLevel {
    None,      // No permission needed (purely informational)
    ReadOnly,  // Read-only filesystem or network access
    Write,     // Write access to the filesystem
    Execute,   // Arbitrary command execution
    Dangerous, // Potentially dangerous (e.g., bypass sandbox)
}
```

## ToolContext

Shared context passed to every tool invocation:

```rust
pub struct ToolContext {
    pub working_dir: PathBuf,
    pub permission_mode: PermissionMode,
    pub permission_handler: Arc<dyn PermissionHandler>,
    pub cost_tracker: Arc<CostTracker>,
    pub session_id: String,
    pub non_interactive: bool,
    pub mcp_manager: Option<Arc<cc_mcp::McpManager>>,
    pub config: Config,
}
```

Key method: `resolve_path(path)` resolves relative paths against the working directory.

## ToolResult

```rust
pub struct ToolResult {
    pub content: String,    // Content sent back to the model
    pub is_error: bool,     // Whether this was an error
    pub metadata: Option<Value>, // Optional structured metadata for TUI
}
```

## Complete Tool Registry

All 33 built-in tools returned by `all_tools()`:

### Shell & Execution

| Tool | Struct | Permission | Description |
|------|--------|------------|-------------|
| `Bash` | `BashTool` | Execute | Run shell commands |
| `PowerShell` | `PowerShellTool` | Execute | Run PowerShell commands (Windows) |
| `Sleep` | `SleepTool` | None | Pause execution for a duration |

## Tool Lookup

```rust
// Find a tool by exact name (case-sensitive)
let tool = find_tool("Bash"); // Some(BashTool)
let none = find_tool("bash"); // None -- case-sensitive
```

<Note>
The `AgentTool` is intentionally excluded from `cc-tools` to avoid circular dependencies. It lives in `cc-query` and is registered separately by the CLI.
</Note>

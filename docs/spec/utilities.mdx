## Core Data Structures

### CircularBuffer

Fixed-size buffer that auto-evicts oldest items:

```typescript
class CircularBuffer<T> {
  constructor(capacity: number)
  add(item: T): void
  addAll(items: T[]): void
  getRecent(count: number): T[]
  toArray(): T[]
  clear(): void
  length(): number
}
```

### QueryGuard

State machine for query lifecycle (`idle` -> `dispatching` -> `running`):

```typescript
class QueryGuard {
  reserve(): boolean         // idle -> dispatching
  cancelReservation(): void  // dispatching -> idle
  tryStart(): number | null  // -> running, returns generation
  end(generation): boolean   // running -> idle
  forceEnd(): void
  get isActive(): boolean
  // useSyncExternalStore compatible
  subscribe: (listener) => () => void
  getSnapshot: () => boolean
}
```

### Signal

Lightweight event signal primitive used ~15x across the codebase:

```typescript
type Signal<Args extends unknown[] = []> = {
  subscribe: (listener: (...args: Args) => void) => () => void
  emit: (...args: Args) => void
  clear: () => void
}
```

## Shell Execution (`Shell.ts`)

The most important utility file. Provides the core shell execution engine:

- **DEFAULT_TIMEOUT**: 30 minutes
- **File mode** (bash): stdout+stderr both go to a single file fd (atomic O_APPEND)
- **Pipe mode** (hooks, `onStdout`): data flows through StreamWrapper to TaskOutput in-memory
- **CWD tracking**: captures `pwd -P` output to temp file after each command
- **Sandbox wrapping**: calls `SandboxManager.wrapWithSandbox()` when enabled
- **Shell selection priority**: `CLAUDE_CODE_SHELL` env > `$SHELL` > zsh > bash

## Buddy (Companion/Tamagotchi) System

A virtual companion that appears as an ASCII art sprite beside the prompt input. Gated behind the `BUDDY` feature flag, planned for April 1-7, 2026.

### Architecture

The system splits companion data into two parts:

| Part | Type | Storage | Description |
|------|------|---------|-------------|
| **Bones** | `CompanionBones` | Never stored (deterministic) | Visual/stat data derived from `hash(userId + SALT)` |
| **Soul** | `CompanionSoul` | Persisted to config | AI-generated name and personality |

### Rolling Process (Seeded PRNG)

Uses Mulberry32 PRNG seeded from `hashString(userId + 'friend-2026-401')`:

1. Rarity tier (weighted random)
2. Species (18 total)
3. Eye style (6 types)
4. Hat (8 types, none if common)
5. Shiny flag (1% probability)
6. Stats (peak/dump/scatter, floor scales with rarity)

### Rarity Distribution

| Rarity | Weight | Stars | Stat Floor |
|--------|--------|-------|------------|
| Common | 60% | 1 star | 5 |
| Uncommon | 25% | 2 stars | 15 |
| Rare | 10% | 3 stars | 25 |
| Epic | 4% | 4 stars | 35 |
| Legendary | 1% | 5 stars | 50 |

### Species (18 Total)

duck, goose, blob, cat, dragon, octopus, owl, penguin, turtle, snail, ghost, axolotl, capybara, cactus, robot, rabbit, mushroom, chonk

Species names are encoded as `String.fromCharCode(...)` to avoid triggering build-time canary string scans.

## Memory Directory (memdir) System

Long-term memory stored as markdown files in `~/.claude/memory/`:

- Memory scanning with relevance scoring
- Auto-consolidation via "dream" service during idle
- Context injection based on relevance
- Per-project and global memory scopes

## Keybindings System

14 files managing keyboard shortcuts:

- Custom keybinding definitions in `~/.claude/keybindings.json`
- Chord bindings (multi-key sequences)
- Context-aware activation (different bindings per mode)
- Default bindings with override support

## Skills System

20 files implementing the skills marketplace:

- Skills are reusable prompt templates with tool configurations
- Bundled skills, user skills, and marketplace skills
- Skill detection and auto-invocation
- Skill change detection for live updates
- Directory-based skill discovery

## Hooks Schema

User-defined hooks for automation:

| Event | Trigger |
|-------|---------|
| `PreToolUse` | Before tool execution |
| `PostToolUse` | After tool execution |
| `Notification` | On notification |
| `Stop` | On turn completion |

Hooks are configured in `settings.json` and executed by the harness.

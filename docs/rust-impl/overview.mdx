# Rust Reimplementation Overview

The Rust codebase at `src-rust/` is a **complete standalone rewrite** of the TypeScript Claude Code CLI in async Rust. It is not an FFI binding layer, not a partial port, and shares no runtime code with the TypeScript implementation.

It re-implements the same tool names and semantics, permission model, CLAUDE.md discovery, auto-compact logic, MCP client, bridge protocol, and cron scheduler -- all in async Rust using the Tokio runtime.

## Clean-Room Methodology

This rewrite follows a strict **spec-only** approach:

- All behavior is derived from behavioral specification documents in `spec/`
- No TypeScript source code was referenced during implementation
- The Rust port produces identical observable behavior to the original

|-------|---------|------|---------|
| `crates/core` | `cc-core` | Library | Shared types, config, permissions, history, hooks |
| `crates/api` | `cc-api` | Library | API client + SSE streaming |
| `crates/tools` | `cc-tools` | Library | All tool implementations (33 tools) |
| `crates/query` | `cc-query` | Library | Agentic query loop, compact, cron scheduler |
| `crates/tui` | `cc-tui` | Library | ratatui terminal UI |
| `crates/commands` | `cc-commands` | Library | Slash command implementations |
| `crates/mcp` | `cc-mcp` | Library | MCP (Model Context Protocol) client |
| `crates/bridge` | `cc-bridge` | Library | Bridge to claude.ai web UI |
| `crates/cli` | `claude-code` | Binary | Binary entry point (`claude`) |
| `crates/buddy` | `cc-buddy` | Library | Tamagotchi companion system |

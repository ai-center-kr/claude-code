## Production Prompt-Shape Surgery

The most revealing technical detail is the engineering Anthropic did to work around a **real production failure mode** with Capybara.

### The Problem

Capybara can **prematurely stop generating** when the prompt shape resembles a turn boundary after tool results. The model gets confused by certain patterns in the message sequence and interprets them as a signal to stop, cutting off its response.

### The Fix

Rather than waiting for a model fix, the team implemented **prompt-shape surgery** -- restructuring the prompt at the application layer to avoid triggering the issue:

| Technique | Purpose |
|-----------|---------|
| Force a safe boundary marker (`Tool loaded.`) | Prevents ambiguous turn boundaries |
| Relocate risky sibling blocks | Moves content that could trigger premature stops |
| Smoosh reminder text into tool results | Maintains generation flow past dangerous points |
| Add non-empty markers for empty tool outputs | Avoids confusing the model with void results |

All of these mitigations are wrapped with **kill-switchable gates** using `tengu_*` prefixed flags, allowing staged rollout and quick rollback.

## Kill-Switchable Gates

Every model-related feature is wrapped in remotely controllable gates:

```
tengu_ultraplan_model     -- ULTRAPLAN model selection
tengu_penguins_off        -- Fast mode kill-switch
tengu_*                   -- Various model-specific behavior flags
```

This architecture allows Anthropic to:
- Roll out new models to internal users first
- A/B test model performance on external users
- Instantly revert if a model causes issues
- Stage prompt-shape surgery independently from model launches

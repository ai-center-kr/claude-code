All UI components are compiled with the React Compiler for automatic memoization. The framework is Ink -- a terminal React renderer using Box/Text primitives with Yoga flexbox layout.

<Note>
Full spec: [`spec/04_components_core_messages.md`](https://github.com/anthropics/claude-code/blob/main/spec/04_components_core_messages.md)
</Note>

## Architecture

Components use these core Ink primitives throughout:

- `Box`, `Text` -- layout and text
- `useInput`, `useTheme`, `useTerminalFocus` -- Ink hooks
- `Ansi`, `RawAnsi`, `NoSelect`, `Link` -- special rendering nodes
- `ScrollBox`, `ScrollBoxHandle` -- scrollable regions

Feature flags are evaluated at compile time via `feature('FLAG_NAME')` from `bun:bundle`. Global state is accessed via `useAppState`, `useSetAppState`.

## Key Top-Level Components

### App.tsx -- Provider Wrapper

The top-level provider that nests all global contexts:

```typescript
type Props = {
  getFpsMetrics: () => FpsMetrics | undefined
  stats?: StatsStore
  initialState: AppState
  children: React.ReactNode
}
// Nesting: FpsMetricsProvider > StatsProvider > AppStateProvider
```

### AutoUpdater.tsx

npm-based auto-updater that polls GCS every 30 minutes for newer versions:

| Behavior | Detail |
|----------|--------|
| Poll interval | 30 minutes |
| Kill switch | `maxVersion` check |
| Routes | Package manager / Native installer / npm |

### BaseTextInput.tsx

Low-level text input shared by `TextInput` and `VimTextInput`:
- `useDeclaredCursor` for terminal cursor placement
- `usePasteHandler` for clipboard paste
- Highlight rendering support

### BashModeProgress.tsx

Renders in-progress bash command UI with streaming output:

```typescript
type Props = {
  input: string              // Command being run
  progress: ShellProgress    // Current shell progress
  verbose: boolean
}
```

### AgentProgressLine Props

| Prop | Type | Description |
|------|------|-------------|
| `agentType` | string | Type identifier of the sub-agent |
| `toolUseCount` | number | Number of tool uses |
| `tokens` | number \| null | Token count |
| `isLast` | boolean | Controls tree connector character |
| `isResolved` | boolean | Whether task is complete |
| `isError` | boolean | Whether task errored |
| `shouldAnimate` | boolean | Whether to animate spinner |

### AutoModeOptInDialog

Contains legally-reviewed description text. Logs telemetry events:
- `tengu_auto_mode_opt_in_dialog_shown`
- `tengu_auto_mode_opt_in_dialog_accept`
- `tengu_auto_mode_opt_in_dialog_accept_default`
- `tengu_auto_mode_opt_in_dialog_decline`
